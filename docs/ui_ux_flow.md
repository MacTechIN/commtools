# UI/UX 흐름

Figma Make로 전체 앱 UI를 먼저 구현할 때는 [figma_make_ui_prompt.md](./figma_make_ui_prompt.md)를 사용합니다. 해당 프롬프트는 대시보드, 목록, 상세, 생성/수정 폼, 설정, 상태 UI, 반응형 화면, 개발 파일명 매핑을 포함합니다.

## 1. 화면 목록

| 화면 | 목적 | 주요 요소 | 우선순위 |
| --- | --- | --- | --- |
| 홈/진입 | 앱 첫 진입 | 대시보드 이동 또는 기본 안내 | P0 |
| 대시보드 | 운영 현황 확인 | 요약 지표, 최근 아이템, 활동 내역 | P0 |
| 아이템 목록 | 데이터 탐색과 관리 | 검색, 필터, 정렬, 테이블, 생성 버튼 | P0 |
| 아이템 상세 | 단일 데이터 확인 | 상세 정보, 상태, 활동, 수정/삭제 액션 | P0 |
| 아이템 생성 | 새 데이터 등록 | ItemForm, 검증 메시지, 저장/취소 | P0 |
| 아이템 수정 | 기존 데이터 변경 | 기존 값 표시, 변경 저장, reset | P0 |
| 설정 | 사용자/앱 설정 | 프로필, 기본 보기, 알림, 위험 액션 | P1 |

## 2. 사용자 흐름

```text
진입
  -> 대시보드
  -> 아이템 목록
  -> 검색/필터/정렬
  -> 아이템 상세
  -> 수정 또는 삭제
  -> 목록 복귀

진입
  -> 아이템 목록
  -> 새 아이템 생성
  -> 저장 성공
  -> 상세 또는 목록 이동

진입
  -> 설정
  -> 프로필/환경 설정 변경
  -> 저장 결과 확인
```

## 3. 화면별 상세

### 홈/진입

- 목적: 사용자를 앱의 기본 업무 화면으로 안내합니다.
- 진입 조건: `/` 접근
- 주요 액션: 대시보드 이동
- 빈 상태: 해당 없음
- 로딩 상태: 앱 초기화 중 표시
- 오류 상태: 앱 초기화 실패 메시지
- 모바일 고려사항: 헤더 높이를 작게 유지합니다.

### 대시보드

- 목적: 전체 운영 상태를 빠르게 확인합니다.
- 진입 조건: `/dashboard` 접근
- 주요 액션: 새 아이템 생성, 최근 아이템 상세 이동
- 빈 상태: 등록된 아이템이 없으면 생성 액션을 안내합니다.
- 로딩 상태: 요약 지표와 목록 skeleton을 표시합니다.
- 오류 상태: 재시도 버튼이 있는 오류 메시지를 표시합니다.
- 모바일 고려사항: 요약 지표를 1열 또는 2열로 재배치합니다.

### 아이템 목록

- 목적: 아이템을 검색, 필터, 정렬하고 관리합니다.
- 진입 조건: `/items` 접근
- 주요 액션: 검색, 필터, 정렬, 상세 이동, 생성, 수정, 삭제
- 빈 상태: 아이템이 없으면 `New item` 버튼을 강조합니다.
- 로딩 상태: 테이블 skeleton 또는 카드 skeleton을 표시합니다.
- 오류 상태: 목록 조회 실패와 재시도 버튼을 표시합니다.
- 모바일 고려사항: 테이블을 카드형 목록으로 전환합니다.

### 아이템 상세

- 목적: 단일 아이템의 정보와 활동 내역을 확인합니다.
- 진입 조건: `/items/[itemId]` 접근
- 주요 액션: 목록으로 돌아가기, 수정, 삭제
- 빈 상태: 해당 없음
- 로딩 상태: 상세 영역 skeleton을 표시합니다.
- 오류 상태: 찾을 수 없음, 권한 없음, 조회 실패를 구분합니다.
- 모바일 고려사항: 액션 버튼을 상단 또는 하단에 고정하지 않고 본문 흐름 안에 배치합니다.

### 아이템 생성/수정

- 목적: 아이템 정보를 입력하고 저장합니다.
- 진입 조건: `/items/new` 또는 `/items/[itemId]/edit` 접근
- 주요 액션: 저장, 취소, 변경 초기화
- 빈 상태: 해당 없음
- 로딩 상태: 수정 화면에서는 기존 값 로딩 상태를 표시합니다.
- 오류 상태: 필드별 검증 오류와 저장 실패 메시지를 표시합니다.
- 모바일 고려사항: 폼은 단일 컬럼으로 표시합니다.

### 설정

- 목적: 사용자와 앱 기본 설정을 관리합니다.
- 진입 조건: `/settings` 접근
- 주요 액션: 프로필 저장, 기본 보기 설정, 알림 설정, 데이터 관리
- 빈 상태: 해당 없음
- 로딩 상태: 설정값 skeleton을 표시합니다.
- 오류 상태: 저장 실패 또는 설정 조회 실패 메시지를 표시합니다.
- 모바일 고려사항: 섹션을 세로로 쌓고 위험 액션은 명확히 분리합니다.

## 4. 공통 UI 정책

- 버튼: 주요 액션은 한 화면에서 하나만 강하게 강조합니다.
- 폼: 필수값과 오류 메시지를 입력 위치 근처에 표시합니다.
- 로딩: 비동기 작업에는 로딩 상태를 제공합니다.
- 빈 상태: 다음 행동을 유도하는 메시지와 액션을 제공합니다.
- 오류: 복구 가능한 안내와 재시도 액션을 제공합니다.
- 삭제: 삭제 전 반드시 `ConfirmDialog`를 표시합니다.
- 상태 표시: 색상과 텍스트를 함께 사용합니다.

## 5. 접근성 체크

- 키보드로 주요 액션에 접근할 수 있어야 합니다.
- 포커스 상태가 시각적으로 보여야 합니다.
- 상태 배지는 색상만으로 의미를 전달하지 않아야 합니다.
- 폼 입력에는 명확한 label이 있어야 합니다.
- 오류 메시지는 해당 필드 근처에 표시되어야 합니다.
- 모바일에서 텍스트와 버튼이 부모 영역을 넘지 않아야 합니다.
