# commtool 개발 문서

이 폴더는 앱 개발에 필요한 요구사항, 설계, 구현, 테스트, 배포 문서를 관리합니다.

## 문서 목록

- [app_dev_def.md](./app_dev_def.md): 원본 앱 개발 정의 문서
- [requirements.md](./requirements.md): 제품 요구사항과 범위
- [functional_spec.md](./functional_spec.md): 기능 상세 명세
- [architecture.md](./architecture.md): 시스템 구조와 기술 방향
- [data_model.md](./data_model.md): 데이터 모델과 저장 구조
- [api_spec.md](./api_spec.md): API 계약
- [ui_ux_flow.md](./ui_ux_flow.md): 화면 구성과 사용자 흐름
- [figma_make_ui_prompt.md](./figma_make_ui_prompt.md): Figma Make UI 생성 프롬프트
- [development_plan.md](./development_plan.md): 개발 단계와 작업 항목
- [test_plan.md](./test_plan.md): 테스트 전략과 체크리스트
- [deployment_ops.md](./deployment_ops.md): 배포와 운영 기준

## 작성 순서

1. `app_dev_def.md`에 앱의 목적, 대상 사용자, 핵심 기능을 먼저 작성합니다.
2. `requirements.md`에서 개발 범위와 우선순위를 확정합니다.
3. `functional_spec.md`, `ui_ux_flow.md`, `data_model.md`, `api_spec.md`를 함께 구체화합니다.
4. `architecture.md`에서 구현 방향을 확정합니다.
5. `development_plan.md`와 `test_plan.md` 기준으로 개발을 진행합니다.
6. 출시 전 `deployment_ops.md`를 점검합니다.
