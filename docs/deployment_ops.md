# 배포와 운영

## 1. 환경

| 환경 | 목적 | URL | 비고 |
| --- | --- | --- | --- |
| local | 개발 | `http://localhost:3000` | 로컬 개발 서버 |
| staging | 검증 | 미정 | 배포 플랫폼 확정 후 설정 |
| production | 운영 | 미정 | 도메인과 호스팅 확정 후 설정 |

## 2. 환경 변수

| 이름 | 설명 | 필수 | 예시 |
| --- | --- | --- | --- |
| `NODE_ENV` | 실행 환경 | Y | `development`, `production` |
| `APP_BASE_URL` | 앱 기본 URL | Y | `http://localhost:3000` |
| `DATABASE_URL` | 데이터베이스 연결 문자열 | 배포 시 Y | `postgres://...` |
| `AUTH_SECRET` | 인증 서명 secret | 인증 도입 시 Y | secret value |
| `LOG_LEVEL` | 로그 레벨 | N | `info` |

## 3. 배포 절차

1. `git status`로 커밋되지 않은 변경을 확인합니다.
2. 테스트를 실행합니다.
3. 빌드를 실행합니다.
4. 환경 변수를 확인합니다.
5. staging에 배포합니다.
6. `/api/health`를 확인합니다.
7. 주요 기능을 검증합니다.
8. production에 배포합니다.

## 4. 롤백 절차

1. 장애 범위를 확인합니다.
2. 마지막 정상 Git 태그 또는 배포 버전을 확인합니다.
3. 마지막 정상 버전으로 되돌립니다.
4. 데이터 변경이 있었는지 확인합니다.
5. 원인과 대응을 기록합니다.

## 5. 모니터링

- 에러 로그: 서버 로그와 API 오류 로그를 확인합니다.
- 성능 지표: 페이지 로딩, API 응답 시간, 오류율을 확인합니다.
- 알림 채널: 운영 방식 확정 후 설정합니다.
- 장애 대응 담당: 프로젝트 소유자 또는 운영 담당자로 지정합니다.

## 6. 릴리스 체크리스트

- [ ] 요구사항 문서가 최신이다.
- [ ] API 명세가 최신이다.
- [ ] UI/UX 흐름이 Figma 결과와 일치한다.
- [ ] 테스트가 통과했다.
- [ ] 빌드가 성공했다.
- [ ] 환경 변수가 설정됐다.
- [ ] `/api/health`가 정상 응답한다.
- [ ] 롤백 방법이 확인됐다.
- [ ] Git 태그 또는 릴리스 버전이 기록됐다.
