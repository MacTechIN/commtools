# 제품 요구사항 문서

## 1. 개요

- 앱 이름: commtool
- 목적: 업무용 핵심 데이터인 `아이템(Item)`을 생성, 조회, 수정, 삭제하고 운영 현황을 대시보드에서 확인합니다.
- 해결하려는 문제: 업무 데이터가 흩어져 있거나 상태 파악이 어려운 상황에서, 사용자가 한 화면에서 목록과 상세 정보를 빠르게 확인하고 필요한 액션을 수행할 수 있게 합니다.
- 주요 사용자: 운영자, 관리자, 팀 멤버
- 성공 기준: 사용자가 대시보드, 목록, 상세, 생성/수정, 삭제, 설정 흐름을 데스크톱과 모바일에서 사용할 수 있습니다.

## 2. 개발 범위

### MVP 포함 항목

- 앱 공통 레이아웃
- 대시보드
- 아이템 목록 조회
- 검색, 필터, 정렬
- 아이템 상세 조회
- 아이템 생성
- 아이템 수정
- 아이템 삭제
- 삭제 확인 다이얼로그
- 설정 화면
- 로딩/빈/오류 상태
- 폼 입력 검증
- 반응형 UI

### MVP 제외 항목

- 실시간 협업
- 고급 권한 그룹 관리
- 외부 서비스 연동
- 결제 기능
- 알림 자동화
- 다국어 지원
- 고급 통계 리포트

### 향후 확장 항목

- 활동 로그 고도화
- 대량 선택/대량 수정
- 데이터 가져오기/내보내기
- 사용자별 권한 정책
- 외부 API 연동
- 알림과 자동화 규칙

## 3. 사용자 유형

| 사용자 | 설명 | 주요 니즈 |
| --- | --- | --- |
| 운영자 | 데이터를 반복적으로 확인하고 관리하는 사용자 | 빠른 검색, 필터, 수정, 상태 확인 |
| 관리자 | 설정과 위험 액션 권한을 가진 사용자 | 삭제, 설정 변경, 데이터 관리 |
| 팀 멤버 | 필요한 데이터를 조회하고 일부 업데이트하는 사용자 | 상세 확인, 상태 변경, 메모 확인 |

## 4. 핵심 사용자 시나리오

1. 사용자는 대시보드에서 전체 아이템 수, 활성/대기 상태, 최근 업데이트를 확인합니다.
2. 사용자는 아이템 목록에서 검색, 필터, 정렬을 사용해 원하는 데이터를 찾습니다.
3. 사용자는 아이템 상세 화면에서 정보를 확인하고 수정 또는 삭제 액션을 수행합니다.
4. 사용자는 새 아이템을 생성하고 저장 성공 메시지를 확인합니다.
5. 관리자는 설정 화면에서 프로필, 기본 보기, 데이터 관리 옵션을 확인합니다.

## 6. 비기능 요구사항

- 성능: 목록 조회와 필터 결과가 빠르게 표시되어야 하며, 로딩 중에는 skeleton 또는 로딩 상태를 제공합니다.
- 보안: 보호된 API는 현재 사용자를 확인하고, 권한 없는 수정/삭제 요청을 차단합니다.
- 접근성: 키보드 포커스, 충분한 색 대비, 명확한 폼 레이블과 오류 메시지를 제공합니다.
- 호환성: 데스크톱 1440px, 태블릿 768px, 모바일 390px 기준 화면을 지원합니다.
- 운영/모니터링: `/api/health`로 기본 상태를 확인하고, 오류 로그 추적을 위한 `requestId`를 사용합니다.

## 7. 제약사항

- 실제 도메인명은 아직 `Item`으로 임시 정의합니다.
- 인증 방식과 데이터베이스는 미정입니다.
- Figma UI 결과에 따라 컴포넌트 세부 구조가 조정될 수 있습니다.
