# 테스트 계획

## 1. 테스트 범위

- 단위 테스트: 검증 스키마, 서비스 함수, 권한 판단 로직
- 통합 테스트: API route와 repository 흐름
- E2E 테스트: 대시보드, 목록, 상세, 생성, 수정, 삭제 흐름
- 수동 테스트: UI/UX, 접근성, 반응형, 배포 전 점검

## 3. 기능별 체크리스트

- [ ] 로그인 사용자를 확인한다.
- [ ] 인증되지 않은 API 요청을 차단한다.
- [ ] 권한 없는 수정/삭제 요청을 차단한다.
- [ ] 관리자만 삭제할 수 있다.

## 4. 배포 전 확인

- [ ] 환경 변수가 누락되지 않았다.
- [ ] 민감 정보가 커밋되지 않았다.
- [ ] 주요 기능이 운영 환경에서 동작한다.
- [ ] `/api/health`가 정상 응답한다.
- [ ] 에러 로그 확인 방법이 있다.
- [ ] Figma UI와 구현 화면의 주요 구조가 일치한다.
