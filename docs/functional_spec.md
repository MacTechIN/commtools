## 2. 기능별 상세

### F-001. 앱 공통 레이아웃

- 목적: 모든 화면에 일관된 내비게이션과 상단 액션을 제공합니다.
- 진입 경로: 모든 앱 화면
- 입력값: 현재 경로, 현재 사용자
- 처리 규칙: 현재 경로에 맞는 메뉴를 활성화합니다.
- 결과값: `AppShell`, `HeaderBar`, `SideNav`, 본문 영역
- 예외 상황: 모바일에서는 사이드바를 drawer로 표시합니다.
- 권한 조건: 로그인 사용자 기준 표시를 기본으로 합니다.

### F-002. 대시보드

- 목적: 운영 현황을 빠르게 파악합니다.
- 진입 경로: `/dashboard`
- 입력값: 아이템 목록 요약, 최근 활동
- 처리 규칙: 전체/활성/대기/오늘 업데이트 수를 계산합니다.
- 결과값: 요약 카드, 최근 아이템, 활동 내역
- 예외 상황: 데이터 없음, 조회 실패
- 권한 조건: 목록 조회 권한 필요

### F-003. 아이템 목록 조회

- 목적: 전체 아이템을 탐색합니다.
- 진입 경로: `/items`
- 입력값: `page`, `searchQuery`, `statusFilter`, `ownerFilter`, `sortKey`, `sortDirection`
- 처리 규칙: 필터와 정렬 조건을 적용한 목록을 표시합니다.
- 결과값: `items`, 페이지 정보
- 예외 상황: 빈 목록, 검색 결과 없음, 조회 실패
- 권한 조건: 목록 조회 권한 필요

### F-004. 아이템 검색

- 목적: 검색어로 원하는 아이템을 찾습니다.
- 진입 경로: `/items`
- 입력값: `searchQuery`
- 처리 규칙: 이름, 설명, 태그에 검색어를 적용합니다.
- 결과값: 검색된 `items`
- 예외 상황: 검색 결과 없음
- 권한 조건: 목록 조회 권한 필요

### F-005. 아이템 필터링

- 목적: 상태, 담당자, 태그 기준으로 목록을 좁힙니다.
- 진입 경로: `/items`
- 입력값: `statusFilter`, `ownerFilter`, `tagFilter`
- 처리 규칙: 선택된 필터를 조합하여 목록을 표시합니다.
- 결과값: 필터링된 `items`
- 예외 상황: 필터 결과 없음
- 권한 조건: 목록 조회 권한 필요

### F-006. 아이템 정렬

- 목적: 목록 순서를 사용자가 원하는 기준으로 변경합니다.
- 진입 경로: `/items`
- 입력값: `sortKey`, `sortDirection`
- 처리 규칙: 최신 수정일을 기본 정렬로 사용합니다.
- 결과값: 정렬된 `items`
- 예외 상황: 정렬 대상 값이 없는 경우 마지막에 표시합니다.
- 권한 조건: 목록 조회 권한 필요

### F-007. 아이템 상세 조회

- 목적: 단일 아이템의 상세 정보를 확인합니다.
- 진입 경로: `/items/[itemId]`
- 입력값: `itemId`
- 처리 규칙: 아이템 존재 여부와 접근 권한을 확인합니다.
- 결과값: `selectedItem`, 활동 내역, 메타데이터
- 예외 상황: 찾을 수 없음, 권한 없음, 조회 실패
- 권한 조건: 상세 조회 권한 필요

### F-008. 아이템 생성

- 목적: 새 아이템을 등록합니다.
- 진입 경로: `/items/new`
- 입력값: `itemInput`
- 처리 규칙: 필수값과 필드 길이를 검증한 뒤 저장합니다.
- 결과값: `createdItem`
- 예외 상황: 검증 실패, 저장 실패
- 권한 조건: 생성 권한 필요

### F-009. 아이템 수정

- 목적: 기존 아이템 정보를 변경합니다.
- 진입 경로: `/items/[itemId]/edit`
- 입력값: `itemId`, `itemInput`
- 처리 규칙: 기존 아이템 조회, 권한 확인, 입력 검증 후 저장합니다.
- 결과값: `updatedItem`
- 예외 상황: 변경사항 없음, 검증 실패, 권한 없음, 저장 실패
- 권한 조건: 수정 권한 필요

### F-010. 아이템 삭제

- 목적: 불필요한 아이템을 제거합니다.
- 진입 경로: 목록 또는 상세 화면
- 입력값: `itemId`
- 처리 규칙: 확인 다이얼로그 승인 후 삭제합니다.
- 결과값: 삭제된 `itemId`
- 예외 상황: 권한 없음, 이미 삭제됨, 삭제 실패
- 권한 조건: 관리자 삭제 권한 필요

### F-011. 설정 관리

- 목적: 사용자와 앱 기본 설정을 관리합니다.
- 진입 경로: `/settings`
- 입력값: 프로필 값, 기본 보기, 알림 설정, 테마 설정
- 처리 규칙: 저장 전 입력값을 검증합니다.
- 결과값: 저장된 설정
- 예외 상황: 저장 실패, 위험 액션 취소
- 권한 조건: 사용자 본인 설정은 가능, 위험 액션은 관리자만 가능

### F-012. 공통 상태 UI

- 목적: 모든 주요 화면에서 상태를 일관되게 표현합니다.
- 진입 경로: 모든 화면
- 입력값: `isLoading`, `errorMessage`, `fieldErrors`
- 처리 규칙: 상태 우선순위는 로딩, 오류, 빈 상태, 기본 상태 순서로 판단합니다.
- 결과값: `LoadingState`, `ErrorMessage`, `EmptyState`, `Toast`, `ConfirmDialog`
- 예외 상황: 여러 상태가 동시에 발생하면 오류 상태를 우선 표시합니다.
- 권한 조건: 없음

## 3. 공통 정책

- 인증/인가: 보호 API는 `requireUser(request)`를 통해 현재 사용자를 확인합니다.
- 입력 검증: 생성/수정 요청은 `itemSchema.ts`의 스키마 기준으로 검증합니다.
- 오류 메시지: 사용자에게 복구 가능한 메시지를 표시합니다.
- 로그 기록: 서버 오류와 주요 변경 요청은 `requestId`와 함께 기록합니다.
- 알림 정책: 생성, 수정, 삭제 성공/실패는 토스트로 표시합니다.

## 4. 상태 정의

| 상태 | 의미 | 전환 조건 |
| --- | --- | --- |
| Default | 정상 데이터 표시 | API 응답 성공 |
| Loading | 데이터를 가져오는 중 | API 요청 시작 |
| Empty | 데이터가 없음 | 목록 응답이 빈 배열 |
| NoResults | 검색/필터 결과 없음 | 조건 적용 후 결과 없음 |
| Error | 요청 실패 | API 오류 또는 네트워크 오류 |
| ValidationError | 입력값 오류 | 스키마 검증 실패 |
| Saving | 저장 중 | 생성/수정/삭제 요청 진행 |
| Success | 작업 성공 | 변경 요청 완료 |

## 5. 엣지 케이스

- 삭제된 아이템 상세 URL로 접근
- 권한 없는 사용자의 삭제 요청
- 검색어는 있지만 결과가 없는 상태
- 저장 중 사용자가 페이지를 벗어나는 경우
- 모바일 화면에서 긴 이름이나 태그가 넘치는 경우
- API 요청 실패 후 재시도하는 경우
