# commtool 앱 기능 정의서

## 1. 문서 목적

이 문서는 `commtool` 앱의 기능 범위와 사용자 흐름을 정의합니다. 개발 계획, UI/UX 설계, API 명세, 데이터 모델, 테스트 계획은 이 문서를 기준으로 구체화합니다.

현재 앱의 핵심 도메인은 임시로 `아이템(Item)`이라고 부릅니다. 실제 서비스 목적이 확정되면 `아이템`, `items`, `Item`, `itemId` 표현을 실제 도메인명으로 교체합니다.

예시:

- 프로젝트 관리 앱이면 `Item` -> `Project`
- 고객 관리 앱이면 `Item` -> `Customer`
- 문서 관리 앱이면 `Item` -> `Document`
- 메시지 관리 앱이면 `Item` -> `Message`

## 2. 앱 개요

`commtool`은 업무용 핵심 데이터를 생성, 조회, 수정, 삭제하고 운영 상태를 한눈에 확인하기 위한 웹 기반 관리 도구입니다. 사용자는 대시보드에서 전체 현황을 확인하고, 목록 화면에서 데이터를 검색/필터링하며, 상세 화면에서 단일 항목의 정보를 검토하고 필요한 액션을 수행합니다.

## 3. 앱 목표

- 핵심 업무 데이터를 빠르게 조회하고 관리할 수 있게 합니다.
- 목록, 상세, 생성, 수정, 삭제 흐름을 명확하게 제공합니다.
- 운영 현황을 대시보드에서 즉시 파악할 수 있게 합니다.
- 입력 검증, 오류 처리, 빈 상태, 로딩 상태를 일관되게 제공합니다.
- 개발자가 문서와 UI를 기준으로 바로 구현할 수 있는 구조를 만듭니다.

## 4. 주요 사용자

| 사용자 유형 | 설명 | 주요 목적 |
| --- | --- | --- |
| 운영자 | 반복적으로 데이터를 확인하고 관리하는 사용자 | 목록 조회, 필터링, 수정, 상태 확인 |
| 관리자 | 사용자 설정과 데이터 관리 권한을 가진 사용자 | 설정 관리, 삭제, 데이터 내보내기 |
| 팀 멤버 | 필요한 데이터를 조회하고 업데이트하는 사용자 | 상세 확인, 상태 변경, 기록 확인 |

## 6. 전체 사용자 흐름

```text
앱 진입
  -> 대시보드 확인
  -> 아이템 목록 이동
  -> 검색/필터/정렬
  -> 아이템 상세 확인
  -> 생성 또는 수정
  -> 저장 결과 확인
  -> 필요 시 삭제
  -> 설정 관리
```

## 7. 화면 정의

| 화면 | 경로 | 목적 | 주요 기능 |
| --- | --- | --- | --- |
| 홈/진입 | `/` | 앱 진입점 | 대시보드 이동 또는 기본 화면 표시 |
| 대시보드 | `/dashboard` | 운영 현황 확인 | 요약 지표, 최근 아이템, 활동 내역 |
| 아이템 목록 | `/items` | 전체 데이터 관리 | 검색, 필터, 정렬, 목록, 생성 진입 |
| 아이템 상세 | `/items/[itemId]` | 단일 데이터 확인 | 상세 정보, 활동, 수정/삭제 액션 |
| 아이템 생성 | `/items/new` | 새 데이터 등록 | 입력 폼, 검증, 저장 |
| 아이템 수정 | `/items/[itemId]/edit` | 기존 데이터 수정 | 기존 값 표시, 검증, 저장 |
| 설정 | `/settings` | 사용자/앱 설정 관리 | 프로필, 환경 설정, 데이터 관리 |

## 8. 기능 상세

### F-001. 앱 공통 레이아웃

- 목적: 모든 화면에서 일관된 탐색과 구조를 제공합니다.
- 주요 요소:
  - `AppShell`
  - `HeaderBar`
  - `SideNav`
  - 본문 영역
  - 사용자 메뉴
- 동작:
  - 현재 화면에 맞는 메뉴 활성 상태를 표시합니다.
  - 데스크톱에서는 좌측 내비게이션을 고정합니다.
  - 모바일에서는 메뉴 버튼으로 내비게이션을 엽니다.

### F-002. 대시보드

- 목적: 앱의 현재 운영 상태를 빠르게 확인합니다.
- 주요 요소:
  - 전체 아이템 수
  - 활성 아이템 수
  - 대기 아이템 수
  - 오늘 업데이트된 아이템 수
  - 최근 아이템 목록
  - 최근 활동 내역
- 상태:
  - 기본 상태
  - 로딩 상태
  - 빈 상태
  - 오류 상태

### F-003. 아이템 목록 조회

- 목적: 사용자가 핵심 데이터를 빠르게 탐색합니다.
- 주요 요소:
  - 목록 테이블
  - 모바일 카드 목록
  - 페이지네이션
  - 행 액션 메뉴
- 표시 필드:
  - 이름
  - 상태
  - 담당자
  - 태그
  - 수정일
  - 액션

### F-004. 아이템 검색

- 목적: 이름, 설명, 태그 등으로 데이터를 빠르게 찾습니다.
- 입력값:
  - `searchQuery`
- 동작:
  - 검색어 입력 시 목록 결과를 갱신합니다.
  - 결과가 없으면 `No search results` 상태를 표시합니다.

### F-005. 아이템 필터링

- 목적: 상태나 담당자 기준으로 목록을 좁힙니다.
- 필터:
  - 상태
  - 담당자
  - 태그
- 주요 변수:
  - `statusFilter`
  - `ownerFilter`
  - `tagFilter`

### F-006. 아이템 정렬

- 목적: 사용자가 원하는 기준으로 목록 순서를 바꿉니다.
- 정렬 기준:
  - 최신 수정일
  - 이름
  - 상태
  - 담당자
- 주요 변수:
  - `sortKey`
  - `sortDirection`

### F-007. 아이템 상세 조회

- 목적: 단일 데이터의 상세 정보를 확인합니다.
- 주요 요소:
  - 아이템 이름
  - 상태 배지
  - 담당자
  - 설명
  - 태그
  - 메타데이터
  - 활동 내역
- 예외 상태:
  - 찾을 수 없음
  - 권한 없음
  - 조회 실패

### F-008. 아이템 생성

- 목적: 새 아이템을 등록합니다.
- 입력 필드:
  - 이름
  - 설명
  - 상태
  - 담당자
  - 태그
  - 메모
- 검증:
  - 이름은 필수입니다.
  - 설명은 최대 길이를 넘을 수 없습니다.
  - 담당자는 유효한 사용자여야 합니다.
- 성공 시:
  - 생성된 아이템 상세 또는 목록으로 이동합니다.
  - 성공 토스트를 표시합니다.

### F-009. 아이템 수정

- 목적: 기존 아이템 정보를 변경합니다.
- 동작:
  - 기존 값을 폼에 표시합니다.
  - 변경사항을 검증합니다.
  - 저장 중에는 저장 버튼을 비활성화합니다.
  - 저장 성공 후 상세 화면으로 이동하거나 성공 메시지를 표시합니다.
- 예외:
  - 저장 실패
  - 권한 없음
  - 변경사항이 없는 상태

### F-010. 아이템 삭제

- 목적: 불필요한 아이템을 제거합니다.
- 동작:
  - 삭제 버튼 클릭 시 확인 다이얼로그를 표시합니다.
  - 사용자가 확인하면 삭제 요청을 보냅니다.
  - 삭제 성공 후 목록으로 이동합니다.
- 안전장치:
  - 삭제 전 확인
  - 위험 액션 스타일 적용
  - 삭제 실패 시 복구 안내 표시

### F-011. 설정 관리

- 목적: 사용자와 앱 기본 설정을 관리합니다.
- 섹션:
  - 프로필
  - 기본 보기 설정
  - 알림 설정
  - 테마 설정
  - 데이터 내보내기
  - 위험 액션 영역
- 상태:
  - 저장 성공
  - 저장 실패
  - 위험 액션 확인

### F-012. 공통 상태 UI

- 목적: 사용자에게 현재 상태를 명확히 전달합니다.
- 상태:
  - `LoadingState`
  - `EmptyState`
  - `ErrorMessage`
  - `Toast`
  - `ConfirmDialog`
  - `PermissionDenied`
  - `NotFound`

| 기능 | Method | Path | 설명 |
| --- | --- | --- | --- |
| 상태 확인 | GET | `/api/health` | 서버 상태 확인 |

## 13. 비기능 요구사항

- 반응형:
  - 데스크톱, 태블릿, 모바일 화면을 지원합니다.
  - 모바일에서는 테이블을 카드형 목록으로 전환합니다.
- 접근성:
  - 키보드 포커스가 보여야 합니다.
  - 색만으로 상태를 전달하지 않습니다.
  - 폼 필드는 명확한 레이블을 가집니다.
- 성능:
  - 목록 조회 중 로딩 상태를 제공합니다.
  - 검색/필터 결과가 빠르게 갱신되어야 합니다.
- 안정성:
  - API 오류 시 재시도 액션을 제공합니다.
  - 삭제 같은 위험 액션은 확인 절차를 거칩니다.
- 보안:
  - 보호된 API는 로그인 사용자를 확인합니다.
  - 권한 없는 삭제/수정 요청을 차단합니다.

## 14. 성공 기준

- 사용자가 대시보드에서 전체 현황을 파악할 수 있습니다.
- 사용자가 아이템을 검색, 필터, 정렬할 수 있습니다.
- 사용자가 아이템을 생성, 상세 조회, 수정, 삭제할 수 있습니다.
- 빈 상태, 로딩 상태, 오류 상태가 모든 주요 화면에 제공됩니다.
- 데스크톱과 모바일에서 핵심 흐름을 사용할 수 있습니다.
- Figma UI와 실제 컴포넌트 구조가 같은 이름 체계를 사용합니다.
